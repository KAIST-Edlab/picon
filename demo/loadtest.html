PICon Frontend Load Test — Popup Coordinator

Before running, three things:
1. Allow popups for this site. First time you click Spawn, the browser will block most of them — look for the popup-blocked icon in the address bar, click "Always allow popups from kaist-edlab.github.io", then Spawn again.
2. On Railway: set MAX_COST_PER_IP_DAILY=0 and optionally bump MAX_CONCURRENT_JOBS. Restore after test. (All popups share this machine's IP, so the daily cap would trip instantly otherwise. There's no other IP-based logic in the backend, so with the cap off, 50-from-1-IP behaves identically to 50-from-50-IPs as far as the API is concerned.)
3. Start small — N=5 first to verify popups, form-fill, and backend all work end-to-end. Then ramp to 10, 20, 50. Each popup is a real top-level browser window consuming real memory; your OS may start thrashing past ~30 visible windows.
